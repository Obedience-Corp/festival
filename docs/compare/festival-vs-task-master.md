# Festival vs Task Master

[Task Master](https://github.com/eyaltoledano/claude-task-master) is an AI-powered task-management system for AI-driven development.

Festival includes task execution, but it is not only a task manager. Its main unit is the mission workspace: related repos, docs, research, intents, plans, lifecycle state, task context, completion criteria, and traceable commits.

## At a glance

| If you mostly need | Better fit |
|---|---|
| AI task decomposition and tracking | Task Master |
| Tasks connected to multi-repo mission context | Festival |
| A task manager for AI-driven development | Task Master |
| A workspace that keeps planning, execution, and commits tied together | Festival |

## Use Task Master when

- the main problem is breaking work into tasks
- you want a task-management layer more than a workspace model
- repo and docs structure are already solved elsewhere

## Use Festival when

- the main problem is keeping tasks connected to the broader mission
- one feature touches code, docs, decisions, and multiple repos
- you want session handoff and commit traceability built into the workflow
