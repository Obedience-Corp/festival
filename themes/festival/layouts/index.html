Reliable Autonomous Execution

Long-running AI coding work you can trust.

Festival is a goal-oriented project planning methodology for long-running AI coding work. A goal breaks down into phases, sequences, and tasks — planned top-down, executed in order. Agents pull their next task from the festival, not from you, so work compounds across sessions without constant re-briefing. Works with Claude Code, Codex, Cursor, OpenCode, Aider, and any agent that can read files and run shell commands.

Why Teams Use Festival

Autonomous AI execution only earns trust when there's a real audit trail. Festival makes the whole loop explicit.

The festival hands off tasks, not you
Agents pull their next task from the festival with `fest next`. No re-briefing, no chat threads, no asking you what to do. You're interrupted as little as possible.
Goals drive the work
Every festival starts from a measurable goal and breaks down into phases, sequences, and tasks — planned top-down, executed in order. Every task has explicit done criteria before an agent starts.
Every commit is auditable
`fest commit` ties code changes back to the task and plan that caused them, so reviews and handoffs are grounded in real traceability.
Work compounds across sessions
Each task builds on the festival's accumulated plans, decisions, and completed work. Nothing starts from zero.
