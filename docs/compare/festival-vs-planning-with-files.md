# Festival vs planning-with-files

[planning-with-files](https://github.com/OthmanAdi/planning-with-files) is a Claude Code plugin that uses persistent Markdown files for planning, progress tracking, and knowledge storage.

Festival shares the same core belief: agent state should live in files, not only chat. Festival adds a CLI-managed workspace for multi-repo project organization, navigation, intents, lifecycle state, execution ordering, validation, and commit traceability.

## At a glance

| If you mostly need | Better fit |
|---|---|
| A lightweight persistent-planning skill | planning-with-files |
| A full workspace for multi-repo AI coding work | Festival |
| Markdown planning inside Claude Code | planning-with-files |
| Multi-repo navigation, execution state, and traceable commits | Festival |

## Use planning-with-files when

- you want a simpler Markdown planning workflow
- your main tool is Claude Code and you prefer a plugin-first setup
- you do not need workspace-level repo management

## Use Festival when

- you want a durable project operating system for AI coding work
- multiple repos, docs, and plans have to stay connected
- you need `camp` and `fest` to provide explicit execution structure
