Why Teams Use Festival

AI coding tools are fast inside one session. Festival is for the part that usually falls apart after that.

Agents lose the mission
New sessions should not have to reconstruct goals, decisions, and next steps from chat history.
Multi-repo work gets scattered
Festival keeps repos, docs, research, and plans in one local workspace that both humans and agents can navigate.
Handoffs restart from zero
`fest next`, lifecycle state, and traceable commits make pause, resume, and handoff much less brittle.

Where Festival Fits

Festival is a local-first planning and context layer. It is not a hosted task manager, a prompt pack, or an agent runtime.

Use Festival when
One mission touches multiple repos, spans multiple sessions, and needs durable next tasks, done criteria, and commit traceability.
Use an orchestrator when
You need to spawn, supervise, and budget many agents. Festival works well under that layer, not instead of it.
Use both together
Let the runtime manage agents while Festival provides the durable mission state they all read and update.
