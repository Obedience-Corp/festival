# Festival vs Agent OS

[Agent OS](https://github.com/buildermethods/agent-os) helps users inject codebase standards and shape better specs so agents stay aligned.

Festival is focused on mission-level project memory: repos, docs, research, plans, tasks, completion criteria, reviews, handoffs, and commits.

## At a glance

| If you mostly need | Better fit |
|---|---|
| Better codebase standards and spec alignment | Agent OS |
| Durable mission state across repos and sessions | Festival |
| Lightweight agent standards | Agent OS |
| A local workspace and execution ledger | Festival |

## Use Agent OS when

- your main problem is making agents follow your standards
- you want spec and repository conventions to be easier for agents to obey

## Use Festival when

- your main problem is keeping the whole mission coherent
- multiple repos and sessions are involved
- you want planning, execution, and commits connected in one place

## They can work together

Agent OS standards can live inside a Festival workspace, and Festival can keep execution tied to a plan.
